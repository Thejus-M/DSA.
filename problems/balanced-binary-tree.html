Balanced Binary Tree.

Easy Tree DFS Recursion
PROB:

Given a binary tree, determine if it is height-balanced.

For this problem, a height-balanced binary tree is defined as a binary tree in which the left and right subtrees of every node differ in height by no more than 1.

CONST:
  • The number of nodes in the tree is in the range [0, 5000].
  • -10^4 <= Node.val <=10^4
EG:
INPUT
root = [3,9,20,null,null,15,7]
OUTPUT
true
INPUT
root = [1,2,2,3,3,null,null,4,4]
OUTPUT
false

Intuition

A binary tree is balanced if the height of its left and right subtrees differ by at most 1, AND both the left and right subtrees are themselves balanced.

This naturally suggests a recursive approach (Depth First Search).

We can write a helper function that returns two values: the height of the tree, and a boolean indicating if it is balanced. We compute this bottom-up.

Solution: Bottom-Up Recursive DFS

Python Solution 
from typing import Optional

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def isBalanced(self, root: Optional[TreeNode]) -> bool:
        def fun(root):
            if not root:
                return 0, True
            
            l, l_bool = fun(root.left)
            r, r_bool = fun(root.right)
            
            # Height is max of left/right + 1
            # Balanced if difference <= 1 AND subtrees are balanced
            return max(l, r) + 1, (abs(r - l) <= 1) and l_bool and r_bool
            
        return fun(root)[1]

Complexity Analysis

Time Complexity
O(n)
We visit every node once.
Space Complexity
O(n)
In the worst case (skewed tree), the recursion stack depth is O(n).
