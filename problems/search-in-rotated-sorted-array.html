Search in Rotated Sorted Array.

Medium Array Binary Search
PROB:

There is an integer array nums sorted in ascending order (with distinct values). Prior to being passed to your function, nums is possibly rotated at an unknown pivot index k (1 <= k < nums.length) such that the resulting array is [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]] (0-indexed).

Given the array nums after the possible rotation and an integer target, return the index of target if it is in nums, or -1 if it is not in nums.

You must write an algorithm with O(log n) runtime complexity.

CONST:
  • 1 <= nums.length <=5000
  • -10^4 <= nums[i] <=10^4
  • All values of nums are distinct.
  • nums is an ascending array that is possibly rotated.
  • -10^4 <= target <=10^4
EG:
INPUT
nums = [4,5,6,7,0,1,2], target = 0
OUTPUT
4
INPUT
nums = [4,5,6,7,0,1,2], target = 3
OUTPUT
-1

Intuition

The array is sorted but rotated. This means if we split the array into two halves, at least one half will always be sorted.

We can use a modified Binary Search. In a standard Binary Search, we check if the target is in the left or right half. Here, we first check which half is sorted.

  • If the left half [left...mid] is sorted, we check if the target lies within this range. If yes, we search left; otherwise, we search right.
  • If the left half is NOT sorted, then the right half [mid...right] MUST be sorted. We check if the target lies within the right sorted range. If yes, we search right; otherwise, we search left.

Solution: Modified Binary Search

Python Solution 
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = (left + right) // 2
            
            if nums[mid] == target:
                return mid
            
            # Check if left half is sorted
            if nums[left] <= nums[mid]:
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            # Otherwise, right half is sorted
            else:
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1
                    
        return -1

Complexity Analysis

Time Complexity
O(log n)
Binary Search splits the search space in half.
Space Complexity
O(1)
No extra space used.
