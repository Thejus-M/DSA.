Time Based Key-Value Store.

Medium Hash Map Binary Search Design
PROB:

Design a time-based key-value data structure that can store multiple values for the same key at different time stamps and retrieve the key's value at a certain timestamp.

Implement the TimeMap class:

  • TimeMap() Initializes the object of the data structure.
  • void set(String key, String value, int timestamp) Stores the key key with the value value at the given time timestamp.
  • String get(String key, int timestamp) Returns a value such that set was called previously, with timestamp_prev <= timestamp. If there are multiple such values, it returns the value associated with the largest timestamp_prev. If there are no values, it returns "".
CONST:
  • 1 <= key.length, value.length <=100
  • key and value consist of lowercase English letters and digits.
  • 1 <= timestamp <=10^7
  • All the timestamps timestamp of set are strictly increasing.
  • At most 2 * 10^5 calls will be made to set and get.
EG:
INPUT
["TimeMap", "set", "get", "get", "set", "get", "get"]
[[], ["foo", "bar", 1], ["foo", 1], ["foo", 3], ["foo", "bar2", 4], ["foo", 4], ["foo", 5]]
OUTPUT
[null, null, "bar", "bar", null, "bar2", "bar2"]
// Explanation:
TimeMap timeMap = new TimeMap();
timeMap.set("foo", "bar", 1); // store the key "foo" and value "bar" along with timestamp = 1.
timeMap.get("foo", 1); // return "bar"
timeMap.get("foo", 3); // return "bar", since there is no value corresponding to foo at timestamp 3 and timestamp 2, then the only value is at timestamp 1 is "bar".
timeMap.set("foo", "bar2", 4); // store the key "foo" and value "bar2" along with timestamp = 4.
timeMap.get("foo", 4); // return "bar2"
timeMap.get("foo", 5); // return "bar2"

Intuition

We need to store values associated with a key and a timestamp. A Hash Map is suitable for mapping keys to values.

Since we need to retrieve values based on timestamps, and the timestamps in set are strictly increasing, we can store the values for each key as a list of pairs [value, timestamp]. This list will naturally be sorted by timestamp.

For the get operation, we need to find the largest timestamp less than or equal to the requested timestamp. Since the list is sorted, we can use Binary Search to find this efficiently.

Solution: HASH MAP + BINARY SEARCH

Python Solution 
from collections import defaultdict

class TimeMap:

    def __init__(self):
        # Dictionary to store key -> list of [value, timestamp]
        self.s = defaultdict(list)

    def set(self, key: str, value: str, timestamp: int) -> None:
        # Append to the list. Since timestamps are increasing, the list remains sorted.
        self.s[key].append([value, timestamp])

    def get(self, key: str, timestamp: int) -> str:
        res = ""
        arr = self.s.get(key, [])

        # Binary Search to find the largest timestamp <= given timestamp
        left, right = 0, len(arr) - 1
        while left <= right:
            m = (left + right) // 2
            if arr[m][1] == timestamp:
                return arr[m][0]
            elif arr[m][1] < timestamp:
                res = arr[m][0] # Candidate answer
                left = m + 1
            else:
                right = m - 1
        return res

Complexity Analysis

Time Complexity
O(1) set, O(log n) get
Set is constant time (append). Get is binary search.
Space Complexity
O(n)
To store all key-value-timestamp entries.
